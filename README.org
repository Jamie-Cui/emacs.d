This repo is a personal GNU/Emacs configuration with two-level deferred loading, Evil mode, and gptel LLM integration. Targets Emacs 30.1+.

** What this config does?
1. Use melpa only.
2. Use emacs version 30.1.
3. Use Maple Mono NF CN (NO Ligature!): [[https://github.com/subframe7536/maple-font][repo]], [[https://github.com/subframe7536/maple-font/releases/download/v7.8/MapleMonoNL-NF-CN-unhinted.zip][download]].
4. Use Plantuml for uml drawing: [[https://github.com/plantuml/plantuml/releases/download/v1.2024.7/plantuml-1.2024.7.jar][download]].
5. Use emacs build-in tree-sitter.

* Architecture

The configuration loads in three layers:

1. *Bootstrap*. ~early-init.el~ runs first (GC tuning, ~package-enable-at-startup~,
   early UI chrome). The repo ~init.el~ is a manifest: it loads the core layer,
   then the feature modules via ~+emacs/load-modules~.
2. *Core* (~lisp/core/~). Foundational mechanics, loaded with ~require~ in a fixed
   order: ~core-vars~ (user variables, ~+emacs~ group), ~core-paths~ (load paths and
   directory helpers), ~core-startup~ (version gate, startup restore, proxy,
   workarounds, server), ~core-package~ (package.el + use-package), ~core-loader~
   (the module loader), ~core-util~ (cross-cutting commands).
3. *Modules* (~lisp/modules/~). One file per capability domain, each providing a
   feature named ~init-<module>~. Language packages live in ~lisp/modules/lang/~,
   loaded by path-like names such as ~lang/cmake~.

Modules load in manifest order; ~keys~ loads last so every command it binds is
already defined. Set ~+emacs/disabled-modules~ (in your installed ~init.el~) to a
list of module names to skip, e.g. ~'("llm" "lang/lean")~.

** Module boundaries

| Module                 | Responsibility                                            |
|------------------------+-----------------------------------------------------------|
| ~os~                     | OS-specific tweaks (Linux/macOS/Windows/WSL), shell env   |
| ~ui~                     | frames, fonts, cnfonts, modeline, dashboard, themes, TTY  |
| ~editor~                 | editing defaults, smartparens, yasnippet, text commands   |
| ~completion~             | vertico/corfu/consult/orderless/marginalia, embark        |
| ~files~                  | dired, tramp, eshell, eat, backups, auto-save, proced     |
| ~project~                | projectile, perspective, envrc                            |
| ~vc~                     | magit, diff-hl                                            |
| ~prog~                   | eglot, flycheck, citre, compilation, treesit              |
| ~lang/*~                 | per-language config (cmake, bazel, markdown, lean, ...)   |
| ~org~                    | Org core, babel, export, plantuml                         |
| ~notes~                  | org-project, journal, roam, deft, agenda hygiene          |
| ~bibliography~           | citar, ebib, bibtex                                       |
| ~latex~                  | auctex, pdf-tools, xenops, git-overleaf (GUI)             |
| ~reading~                | elfeed                                                    |
| ~llm~                    | gptel, agent-shell, magent, magit-gptel                   |

Local and forked packages live in ~site-lisp/~ and keep their own ~init-config-*~
helper files in ~lisp/~.

* Getting started

** Install

#+begin_src bash
# Install the bootstrap files into ~/.emacs.d (existing files are kept).
make init
# Overwrite existing ~/.emacs.d/{init,early-init}.el instead:
make init-force
#+end_src

~make init~ installs ~templates/early-init.el~ and ~templates/init.el~, substituting
this checkout's path for the ~@REPO_DIRECTORY@~ placeholder. Machine-local
settings (proxy, org root, theme, package archives) belong in the installed
~~/.emacs.d/init.el~.

Useful checks: ~make smoke~ (offline startup test of both chains) and
~make compile~ (byte-compile core + modules to a temp dir).


** Prerequisite
- gcc toolchain (gcc, g++, cmake, makefile, autotools, ...): compile environment
- libgccjit: for emacs's native compilation feature
- libsqlite3: for org-roam database
- libtree-sitter (tested with version 0.25.8, [[https://github.com/tree-sitter/tree-sitter][repo]]): for tree-sitter,
- (optional) texlive: for latex
- (optional) direnv: for separating project envs (~sudo dnf install direnv~)

#+begin_src bash
# macos, if you have not done so, in ~/.zshrc
export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/opt/homebrew/lib/
# why tree-sitter@0.25? see: https://github.com/jdtsmith/emacs-mac/issues/121
brew install autoconf texinfo libgccjit gcc gnutls tree-sitter@0.25 pkg-config coreutils
#+end_src

** Download and Compile Emacs

#+begin_src bash
# download
wget -c https://github.com/emacs-mirror/emacs/archive/refs/tags/emacs-30.1.tar.gz
# or the following dev branch with tty-child-frame feature
wget -c https://github.com/emacs-mirror/emacs/archive/refs/heads/scratch/tty-child-frames.zip

# unzip and enter emacs folder
./autogen.sh
# (opt) configure terminal emacs
./configure --with-native-compilation --with-tree-sitter --with-gnutls --without-x-toolkit --without-xpm --without-gif --without-tiff --with-sqlite3 --without-pop --without-mailutils
# (opt) configure gui emacs with tiling window manager (WARNING: fcitx will not be avaliable inside emacs)
./configure --with-native-compilation --with-tree-sitter --with-sqlite3 --with-rsvg --without-pop --without-mailutils --without-x-toolkit
# (opt) configure gui emacs with pgtk
./configure --with-native-compilation --with-tree-sitter --with-sqlite3 --with-rsvg --without-pop --without-mailutils --with-pgtk
# (opt) configure gui emacs with default x toolkit (for MacOS)
./configure --with-native-compilation --with-tree-sitter --with-sqlite3 --with-rsvg --without-pop --without-mailutils --with-x-toolkit
# compile
make -j
# install
sudo make install
#+end_src

MacOS: libgccjit failed to compile and run? see: https://github.com/d12frosted/homebrew-emacs-plus/issues/680#issuecomment-2103717530

Have tree-sitter problem? see: [[https://www.reddit.com/r/emacs/comments/1e57kzy/a_solution_for_getting_emacs_to_compile_with/?utm_source=share&utm_medium=web3x&utm_name=web3xcss&utm_term=1&utm_content=share_button][link]]. Or download tree-sitter pre-built binaries [[https://github.com/emacs-tree-sitter/tree-sitter-langs][here]].
